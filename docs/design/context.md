This project is the infrastructure for Radspion, a secret agency with an undisclosed purpose.  This is good because it gives me the flexibility to create different story lines for missions.

My inspiration for this system comes from many sources:

* The movie National Treasure - I love that they kept coming back to the Declaration of Independence, having to view it different ways to find the solution to the next step.
* Escape Rooms - You have multiple puzzles in one space, and you discover or assemble pieces of the puzzles in whatever order occurs.  Everything is planned so that you can't solve the "final" puzzle until you have solved all the previous puzzles.
* The Westing Game (book) by Ellen Raskin - I love the aspect that it wasn't what they had, it's what they didn't have.  They kept finding words, eventually realized it was American The Beautiful, and then eventually realized that it was the words that never found that mattered.
* Ready Player One (book) by  Ernest Cline - I love the idea of things hidden in places that you have to figure out where, and then you have to figure out the puzzle to get the thing.  I love the idea of the leaderboard celbrating that someone figured out a part of the puzzle (and challenging others to catch up).
* he Adventures of the Black Hand Gang (book) by Hans Jürgen Press - I love that the story unfolded across one-page sections.  The top of the page verifies the solution to the last puzzle, and the rest of the page introduces the next puzzle.  I enjoyed finding the solution in the puzzle (although this aspect will be used minimally in Radspion).

In Radspion, there will be multiple story lines.  Each one can use portions of these inspirational books/movies/games.  Clearance codes gate access to missions; agents submit data to complete fieldwork.  The briefs of all the missions will form the storyline.  I will also provide links to training activities - knowledge they need to solve the missions.  I will provide a HINTS section that will help them if they get stuck.

The puzzles will all be CS and tech-related.  For example, find the solution in the meta data of an image file; look in a branch of a git repository; find covert headers in the HTTP response messages from a server (I plan to use the radspion server itself at some point); As a comment in the HTML served up; In a dot file on a server.

The puzzles will chain togehter.  For example, in one mission the students find the name of a server.  In another they find the SSH key.  In a third they find the username.  Once they have all three, they can SSH to the server and find a hiddle file on the server.  Now that they have access to the server, I can hid files in various places for them to find.  etc.

This provides multiple things to design:

* technical challenges for them to solve - I'm good with this one
* A storyline and characters to weave this into a story
* A large puzzle for them to solve

The 2nd and 3rd items go together, but they are subtly different.  The story is ultimately just window dressing on top of the other 2.  We could swap out characters and reasons to solve the puzzle, but the underlying puzzle is unchanged.

We also could swap out the technical challenges - For most of the places I can hid information, it doesn't matter what I put there.  This means I could use the same puzzle with different challenges - or change the order of the challenges each year based on how I teach a course.
